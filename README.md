## 🚀 Quick Start

### Prerequisites
- Azure CLI (`az --version`)
- kubectl (`kubectl version`)
- Helm (`helm version`)
- Docker (`docker --version`)

### Deploy Everything

```bash
# 1. Create AKS cluster & deploy microservices
cd task1/ && chmod +x infrastructure/setup-aks.sh && ./infrastructure/setup-aks.sh

# 2. Set up CI/CD (ArgoCD)
cd ../task2/ && chmod +x argocd/setup-argocd.sh && ./argocd/setup-argocd.sh

# 3. Install observability stack
cd ../task3/ && chmod +x setup-monitoring.sh && ./setup-monitoring.sh

# 4. Apply security hardening
cd ../task4/ && chmod +x setup-security.sh && ./setup-security.sh
```

### Access Services
```bash
# Application
kubectl get svc -n ingress-nginx    # Get EXTERNAL-IP
# http://<EXTERNAL-IP>/              → Dashboard
# http://<EXTERNAL-IP>/api/health    → API Health

# Grafana (admin / DodoGrafana2026!)
kubectl port-forward svc/monitoring-grafana -n monitoring 3000:80

# ArgoCD (admin / <auto-generated>)
kubectl port-forward svc/argocd-server -n argocd 8080:443

# Jaeger
kubectl port-forward svc/jaeger-query -n monitoring 16686:16686
```

## 💡 Design Principles

1. **Defense-in-depth** — Multiple overlapping security layers
2. **GitOps** — Git as single source of truth for infrastructure
3. **Observability-first** — Metrics, logs, and traces from day one
4. **Least privilege** — Minimal permissions at every layer
5. **Production-grade** — Resource limits, health probes, PDBs, HPA on all services
